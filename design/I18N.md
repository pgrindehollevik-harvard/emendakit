# EmendaKit · Internationalization (NO + EN)

The site ships in two languages from day one. Norwegian (`nb-NO`) is the primary market; English (`en-US`) is for export buyers and for LLM citation in English clinical Q&A.

## URL strategy

Norwegian is the default — no locale prefix. English lives under `/en/`.

Product slugs use the brand name (`emendakit-hospital`, `emendakit-dentist`) — same in both languages, so URLs are stable across locales.

| Route                                 | Norwegian (default)                      | English                                  |
|---------------------------------------|------------------------------------------|------------------------------------------|
| Homepage                              | `/`                                      | `/en/`                                   |
| EmendaKit Hospital                    | `/produkt/emendakit-hospital`            | `/en/product/emendakit-hospital`         |
| EmendaKit Dentist                     | `/produkt/emendakit-dentist`             | `/en/product/emendakit-dentist`          |
| Behovstilpasset / Custom kit          | `/tjenester/behovstilpasset`             | `/en/services/custom-kit`                |
| Opplæring / Training                  | `/opplaering`                            | `/en/training`                           |
| Om oss / About                        | `/om-oss`                                | `/en/about`                              |
| Kontakt / Contact                     | `/kontakt`                               | `/en/contact`                            |

Already configured in `astro.config.mjs`:

```js
i18n: {
  defaultLocale: "no",
  locales: ["no", "en"],
  routing: { prefixDefaultLocale: false },
}
```

## Language switch — UI

The toggle appears in **two places**, always visible:

1. **Header (primary)** — segmented pill `[ NO | EN ]` in the sticky header, just before the "Be om tilbud" CTA. Mono type for visual neutrality. Active state is a navy pill.
2. **Footer (redundant)** — discreet text link `Språk · Language: Norsk / English` in the colophon row, for users who scroll past without noticing the header.

Both controls navigate to **the equivalent URL in the target language** — never just to the localized homepage. If a visitor is on `/produkt/anafylaksiskrin-tannlege` and clicks `EN`, they land on `/en/product/anaphylaxis-kit-dental`, not `/en/`.

## Implementation pattern (Astro)

Every page exports its `pathPairs` so the `<LangSwitch>` component can resolve the equivalent URL:

```astro
---
// src/components/LangSwitch.astro
const { current } = Astro.props;        // "no" | "en"
const { altUrl } = Astro.props;         // resolved at page level
---
<div class="lang-toggle" role="group" aria-label="Velg språk · Choose language">
  <a href={current === "no" ? Astro.url.pathname : altUrl}
     class={current === "no" ? "on" : ""} aria-current={current === "no"} hreflang="nb-NO">NO</a>
  <a href={current === "en" ? Astro.url.pathname : altUrl}
     class={current === "en" ? "on" : ""} aria-current={current === "en"} hreflang="en-US">EN</a>
</div>
```

Each MDX content file declares its alt-language counterpart via frontmatter:

```yaml
---
title: "Anafylaksiskrin / Tannlege"
slug: "anafylaksiskrin-tannlege"
locale: "no"
altSlug: "anaphylaxis-kit-dental"
---
```

The page resolves `altUrl` from `altSlug` + the `/en/` prefix.

## Persistence

- Set a `pref-lang` cookie on click (1-year expiry, SameSite=Lax). Subsequent visits to `/` redirect to `/en/` if the cookie says `en`.
- On first visit, **do not** auto-redirect based on `Accept-Language` — surface the toggle prominently, let the user choose. (Auto-redirect breaks SEO and frustrates bilingual Norwegians.)
- Honor `?lang=en` and `?lang=no` query params as overrides for direct linking from emails, etc.

## SEO / AEO

- Every page emits `<link rel="alternate" hreflang="nb-NO" href="...">` and `<link rel="alternate" hreflang="en-US" href="...">` plus `<link rel="alternate" hreflang="x-default" href="https://www.emendakit.com/">`.
- `sitemap.xml` lists both language URLs under `<xhtml:link>` entries (Astro's `@astrojs/sitemap` integration handles this when `i18n` is configured — already wired up).
- `/llms.txt` summary is **bilingual** (NO followed by EN section). LLM crawlers ingest both.
- Schema.org JSON-LD is emitted in the page's locale; product names, descriptions, and FAQ text are localized.

## Translation workflow

Content is authored in Norwegian first (Kjersti's voice). English versions live alongside as separate MDX files in `src/content/<collection>/` with the `.en.mdx` suffix.

For the initial build, an agent in Conductor can draft English versions from the Norwegian source — but **every clinical claim, dosage, and protocol reference must be reviewed by Kjersti before shipping** to the English site. Add a `reviewed: true` frontmatter field that gates EN pages from the production build until set to `true`.

```yaml
---
locale: "en"
reviewed: false   # blocks EN page from production build
---
```

## Voice notes

- Norwegian: precise, clinical, lightly informal where appropriate — Kjersti and Gregers's voice. Use bokmål, not nynorsk.
- English: neutral international clinical English. Avoid US-specific dosage units. Use ml, mg, IM/IV — not ounces or proprietary trade names.
- Both languages: **no marketing fluff.** Clinicians smell it from a kilometer away.
